# ADR-001: معماری Core-first برای اکوسیستم Yekta SMS

## وضعیت
Accepted

## زمینه
پروژه‌ی Yekta SMS باید چند gateway و چند integration را بدون coupling سخت پشتیبانی کند.  
اگر منطق مشترک در pluginهای جانبی پخش شود، نگهداری، تست و توسعه‌ی آینده دشوار می‌شود.

## تصمیم
- تمام قراردادهای عمومی در `core` تعریف می‌شوند.
- `gateway`ها فقط provider adapter هستند.
- `integration`ها فقط event binding و orchestration سمت محصول ثالث هستند.
- هیچ plugin جانبی نباید منطق shared را برای خودش بازتعریف کند.

## پیامدها
### مثبت
- افزودن gateway جدید ساده‌تر می‌شود
- افزودن integration جدید ساده‌تر می‌شود
- تست‌پذیری افزایش می‌یابد
- observability متمرکزتر می‌شود

### منفی
- طراحی اولیه‌ی `core` باید دقیق‌تر انجام شود
- هر تغییر عمومی نیازمند دقت بیشتری در versioning و documentation است

## نتیجه
این تصمیم مبنای تمام توسعه‌های بعدی در این monorepo است.
